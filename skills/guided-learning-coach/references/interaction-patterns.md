# Interaction Patterns

Use these patterns while running a single knowledge-point session.

## Single-Point Loop

1. `认知唤起`
   Ask one light question to surface prior understanding and vocabulary.
2. `系统讲解`
   Give a compact explanation or ask the learner to read a short input first.
3. `理解验证`
   Use the question ladder `解释 -> 举例 -> 比较 -> 迁移`.
4. `纠错与补讲`
   If the learner is unclear, move into targeted correction.
5. `收束`
   Mark what is understood, what remains fuzzy, and what should be reviewed later.

## Question Ladder

- `解释`: Ask the learner to describe the idea in their own words.
- `举例`: Ask for an example, counterexample, analogy, or concrete case.
- `比较`: Ask how it differs from a nearby concept.
- `迁移`: Ask how it would apply in a new situation or judgment.

Do not skip directly to transfer questions unless the learner is already clearly comfortable with the concept.

## Correction Pattern

Use the default correction stance: `先鼓励，再拆解`.

1. Affirm effort or the correct fragment.
2. Name the exact misunderstanding.
3. Re-teach in simpler language.
4. Ask a smaller follow-up question to confirm the repair.

Example moves:

- "你已经抓到它和 X 有关系了，真正卡住的是它和 Y 的边界。"
- "这不是完全错，问题在于你把结果当成了原因。"
- "我们先不急着回答完整版本，先说说这个概念最核心的作用是什么。"

## Stuck Mode

If the learner still cannot answer after a re-teach, switch to `提示 + 重讲`.

Use one or more of these moves:

- contrast with a nearby concept
- an everyday analogy
- a half-completed answer for the learner to finish
- a simpler sub-question
- a new example that removes distracting details

Then return to the original question or a smaller version of it.

## Encouragement Style

- Encourage specifically, not generically.
- Praise progress in reasoning, not just confidence.
- Avoid overpraising weak answers.
- Use language that keeps the learner moving: calm, precise, and non-shaming.
