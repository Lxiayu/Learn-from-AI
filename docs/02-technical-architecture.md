## 一、架构概述

### 1.1 设计原则
- **高可用性**：99.9%服务可用性，依托CloudBase云原生架构
- **高扩展性**：自动弹性扩缩容，应对用户增长
- **高性能**：响应时间 < 2秒，本地数据优先
- **安全性**：CloudBase内置安全防护，数据加密，权限控制
- **可维护性**：免运维架构，一站式管理控制台
- **成本优势**：按需付费，降低运营成本
- **离线能力**：本地存储保障核心功能离线可用
- **用户自主**：用户可选择数据是否上云，本地优先

### 1.4 混合存储架构核心策略

#### 1.4.1 数据存储策略
- **本地优先**：所有核心数据优先存储在本地
- **云端备份**：用户可选择是否同步数据到云端
- **双重备份**：本地和云端各保存一份数据
- **容错机制**：本地数据破损时自动切换到云端

#### 1.4.2 数据访问策略
- **优先本地**：查看报告、图表等优先使用本地数据
- **本地验证**：检查本地数据完整性和时效性
- **云端降级**：本地数据不可用时使用云端数据
- **智能切换**：根据网络状态和数据状态自动选择

## 二、服务架构设计

### 2.1 服务部署模式

#### 2.1.1 云托管部署（长驻服务）
**适用场景**：需要持续运行的HTTP服务，如用户服务、学习服务等

**技术方案**：
- 运行环境：CloudBase云托管
- 框架：FastAPI
- 部署方式：Docker容器化部署
- 扩缩容：自动弹性扩缩容，根据负载自动调整实例数
- 高可用：多实例部署，自动故障转移

**服务列表**：
1. **用户服务**：用户账号、认证授权、个人信息管理
2. **学习服务**：学习路线、知识点管理、学习记录
3. **复习服务**：间隔复习、复习计划、掌握度评估
4. **数据服务**：学习统计、成就系统、数据导出
5. **同步服务**：数据同步、冲突解决、备份恢复

#### 2.1.2 云函数部署（无服务器）
**适用场景**：事件驱动的短任务，如AI对话、消息推送、定时任务等

**技术方案**：
- 运行环境：CloudBase云函数
- 触发方式：HTTP触发、定时触发、事件触发
- 计费方式：按调用次数和运行时间计费
- 扩缩容：自动弹性扩缩容，无需管理服务器

**云函数列表**：
1. **AI对话函数**：处理苏格拉底式对话、问题生成
2. **消息推送函数**：发送复习提醒、成就通知
3. **数据统计函数**：定时计算学习统计数据
4. **数据清理函数**：定期清理过期数据
5. **同步协调函数**：处理数据同步冲突

### 2.4 同步策略

#### 2.4.1 同步触发时机
- **应用启动时**：检查是否有需要同步的数据
- **网络恢复时**：WiFi恢复时自动同步
- **用户手动触发**：用户主动点击同步按钮
- **定期自动同步**：按设定间隔自动同步
- **数据变更时**：重要数据变更后立即同步

#### 2.4.2 同步流程
```
1. 检查同步设置（是否启用同步）
2. 检查网络状态（是否仅WiFi同步）
3. 获取本地未同步数据
4. 获取云端最新数据
5. 对比数据版本
6. 处理冲突
7. 双向同步
8. 更新同步状态
9. 通知用户同步结果
```

#### 2.4.3 冲突解决策略
| 冲突类型 | 解决策略 | 说明 |
|---------|---------|------|
| 时间戳冲突 | 最新优先 | 以最新更新的数据为准 |
| 内容冲突 | 用户选择 | 重要冲突由用户决定 |
| 删除冲突 | 保留确认 | 需要用户二次确认 |
| 学习进度冲突 | 合并+提示 | 合并进度并提示用户 |

#### 2.4.4 增量同步
- **变更追踪**：每个数据项记录更新时间和同步状态
- **差异计算**：只同步变更的数据，减少网络传输
- **批量处理**：批量同步多个数据项
- **断点续传**：支持同步中断后继续

### 2.5 数据完整性验证

#### 2.5.1 本地数据验证
- **完整性检查**：定期检查本地数据完整性
- **校验和**：使用SHA-256校验和检测数据破损
- **数据修复**：发现破损时从云端同步修复
- **自动修复**：小问题自动修复，大问题提示用户

#### 2.5.2 数据恢复流程
```
1. 检测到数据破损
2. 检查是否有云端备份
3. 提示用户是否恢复
4. 从云端同步修复数据
5. 验证修复后的数据
6. 通知用户恢复结果
```

## 十二、混合存储架构优势总结

### 12.1 用户体验优势
- **离线可用**：无网络环境下仍可使用核心功能
- **极速响应**：本地数据访问速度远超网络请求
- **数据控制**：用户对个人数据有完全控制权
- **灵活选择**：用户可根据需求决定是否上云

### 12.2 技术优势
- **规避限制**：不依赖CloudBase的数据库特性
- **降低依赖**：减少API调用，降低成本
- **容错机制**：本地数据破损时自动切换到云端
- **灵活扩展**：可根据需要添加新的本地存储功能

### 12.3 数据安全优势
- **双重备份**：本地和云端各有一份数据
- **隐私保护**：敏感数据可只存储在本地
- **数据恢复**：云端数据可作为备份恢复来源
