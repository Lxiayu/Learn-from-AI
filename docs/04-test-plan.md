#### 3.5.1 安全测试项
| 测试项 | 测试内容 | 预期结果 |
|--------|---------|---------|
| SQL注入 | 输入SQL语句 | 拦截并报错 |
| XSS攻击 | 输入脚本代码 | 转义处理 |
| CSRF攻击 | 伪造请求 | 拦截并报错 |
| 暴力破解 | 连续错误登录 | 账号锁定 |
| 越权访问 | 访问他人数据 | 拒绝访问 |

| 会话劫持 | 窃取Token | Token失效 |

#### 3.5.2 安全测试用例
```python
class TestSecurity:
    """安全测试"""
    
    def test_sql_injection(self, client):
        """测试SQL注入"""
        response = client.post(
            '/api/v1/auth/login',
            json={
                'email': "admin'--",
                'password': 'anything'
            }
        )
        assert response.status_code == 400
    
    def test_xss_attack(self, client, auth_header):
        """测试XSS攻击"""
        response = client.post(
            '/api/v1/learning/routes',
            json={'topic': '<script>alert("XSS")</script>'},
            headers=auth_header
        )
        assert '<script>' not in response.json()['data']['topic']
    
    def test_unauthorized_access(self, client):
        """测试未授权访问"""
        response = client.get('/api/v1/users/me')
        assert response.status_code == 401
    
    def test_brute_force_protection(self, client):
        """测试暴力破解保护"""
        for i in range(10):
            response = client.post(
                '/api/v1/auth/login',
                json={'email': 'test@test.com', 'password': 'wrong'}
            )
        assert response.status_code == 429  # Too Many Requests
```
